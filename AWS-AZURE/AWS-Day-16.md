## Objective
The objective of Day 16 was to create a new IAM (Identity and Access Management) user in AWS. This task focused on understanding how AWS manages identities and access control at a foundational level.

---

## AWS Services / Concepts Covered
- AWS Identity and Access Management (IAM)
- IAM Users
- Global AWS services
- Least privilege principle

---

## Steps I Took

1. Logged into the AWS Management Console using the provided credentials.
2. Navigated to **IAM (Identity and Access Management)** from the AWS services menu.
3. Opened the **Users** section in the IAM dashboard.
4. Clicked **Create user** to add a new IAM user.
5. Entered the user name **iamuser_mariyam**.
6. Chose to create the user **without attaching any permissions** (permissions can be added later).
7. Reviewed the details and completed the user creation.
8. Verified that the user **iamuser_mariyam** appeared in the IAM users list and was active.

---

## Issues Faced
- No issues were encountered during the creation of the IAM user.

---

## Key Takeaways

- IAM is a core AWS service used to control access to AWS resources.
- IAM users represent individual people or applications.
- IAM is a **global service**, not tied to a specific region.
- Users can be created first and granted permissions later.
- Following the least privilege principle improves security.

---

## Conclusion
Day 16 focused on creating an IAM user, reinforcing the importance of identity management in AWS. This task highlighted how AWS separates identity creation from permission assignment, allowing flexible and secure access control as infrastructure grows.
